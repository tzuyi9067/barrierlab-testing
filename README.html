How to run it locally

Any static server works. No build step, no dependencies. From this folder:

python3 -m http.server 8000
# then open http://localhost:8000

You can also double-click index.html to open it in a browser — the cart, nav, and dropdown all work from file://.

Design system

Palette

  • --purple #6B5BFF — primary CTA and brand accent
  • --purple-deep #4A3FCC — hover and emphasis
  • --lavender #EEEBFF — soft tint for cards and chips
  • --lavender-deep #DCD5FA — stronger periwinkle for gradients
  • --cream #FFFFFF — primary background (pure white)
  • --ink #1A1A1A — primary text, dark sections

Typography

Inter (sans) for everything. Fraunces italic is reserved — it appears only in: the wordmark, <em> inside h1 / h2 / h3, and decorative stat numbers. Never in buttons, product names, small copy, or thumbnails.

Cart architecture

One source of truth: the CATALOG object in assets/shared.js. Each entry maps 1:1 to a future Shopify product.

CATALOG = {
  'bl-01': { code, name, tag, shortTag, price, subPrice, href, viz, initial },
  'bl-02': { ... },
  'bl-03': { ... },
  'bl-bundle-adult': { ... },
  'bl-bundle-family': { ... },
}

Cart state lives in localStorage at key barrierlab:cart:v1 — an array of { sku, qty, subscribe }. Any DOM change fires a barrierlab:cart-updated custom event so cart.html and checkout.html stay in sync.

Wiring "Add to cart"

Put class="js-add-to-cart" and data-sku="bl-01" on any button. Add data-subscribe="true" to route to the subscription price. That's it — shared.js auto-wires them on every page.

Porting to Shopify

The site is built to swap into Shopify cleanly. Nothing here holds you hostage.

Three steps:

  • Products: Create five products in Shopify admin with handles matching the SKUs (bl-01, bl-02, bl-03, bl-bundle-adult, bl-bundle-family). Prices and subscription pricing are already set in CATALOG — copy them over.
  • Cart routing: Replace the localStorage cart in shared.js with Shopify's AJAX Cart API (/cart/add.js, /cart.js). Keep the same function signatures (addToCart, loadCart, setQty, removeItem) — every page already calls through these.
  • Checkout: Replace the "Place mock order" button in checkout.html with a redirect to Shopify's /checkout. The rest of the demo checkout can be deleted; it exists to show you what a styled checkout looks like.

Subscriptions: connect via Recharge, Shopify Subscriptions, or Appstle. The subPrice field in CATALOG corresponds to the subscription price you'll set in your chosen app — 15% off by default.

Adding a new page

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Your page — Barrier Lab</title>
  <link rel="icon" type="image/png" href="assets/mark-purple.png">
  <link rel="stylesheet" href="assets/shared.css">
</head>
<body>
  <section>
    <div class="container">
      <!-- your content -->
    </div>
  </section>
  <script src="assets/shared.js"></script>
</body>
</html>

Nav, footer, and cart drawer inject automatically. Don't hand-roll them.
